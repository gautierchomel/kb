About

Version: WCAG 2.0 and up
Number: 4.1.1
Level: A
Applicability:
  • All Elements
Requirement:

In content implemented using markup languages, elements have complete start and end tags, elements are nested according to their specifications, elements do not contain duplicate attributes, and any IDs are unique, except where the specifications allow these features.
Guidance:
  • Understanding Success Criterion 4.1.1
  • How to meet Success Criterion 4.1.1
Techniques:
  • G134: Validating Web pages
  • G192: Fully conforming to specifications
  • H74: Ensuring that opening and closing tags are used according to specification
  • H75: Ensuring that Web pages are well-formed
  • H88: Using HTML according to spec
  • H93: Ensuring that id attributes are unique on a Web page
  • H94: Ensuring that elements do not contain duplicate attributes

Purpose

The purpose of this success criterion is to ensure that critical markup errors do not cause user agents and assistive technologies to interpret documents differently.

How to Meet

EPUB Creators should use a validator, such as epubcheck, to validate their content. Although not all the issues flagged by a validator must be fixed to meet this success criterion, they will normally flag the major issues that result in failures.

Note that this requirement is closer to ensuring all XML content is well-formed than that it fully validates to its respective standard.

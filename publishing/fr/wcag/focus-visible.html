About

Version: WCAG 2.0 and up
Number: 2.4.7
Level: A
Applicability:
  • Buttons
  • Controls
  • Inputs
Requirement:

Any keyboard operable user interface has a mode of operation where the keyboard focus indicator is visible.
Guidance:
  • Understanding Success Criterion 2.4.7
  • How to meet Success Criterion 2.4.7
Techniques:
  • C15: Using CSS to change the presentation of a user interface component when it receives focus
  • G149: Using user interface components that are highlighted by the user agent when they receive focus
  • G165: Using the default focus indicator for the platform so that high visibility default focus indicators will carry over
  • G195: Using an author-supplied, highly visible focus indicator
  • SCR31: Using script to change the background color or border of the element with focus
  • Using a change of contrast ratio to provide a visible focus indicator (undocumented)

Purpose

The purpose of this success criterion is to ensure that keyboard users are aware which control they are interacting with in a user interface.

How to Meet

Authors need to ensure that there is a visual indicator for the control that currently has focus. This indicator often takes the form of a border or shading around the control.
